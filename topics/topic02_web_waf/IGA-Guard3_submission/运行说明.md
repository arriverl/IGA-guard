# IGA-Guard 3.0 系统运行说明

> 版本：3.0.0 · 适用赛题：Web 攻击载荷混淆逃逸检测

---

## 1. 环境要求

| 项目 | 最低要求 | 推荐 |
|------|----------|------|
| 操作系统 | Windows 10/11、Ubuntu 20.04+、macOS 12+ | Ubuntu 22.04 |
| Python | 3.10 ~ 3.13 | 3.11 |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 2 GB（含模型） | 5 GB |
| GPU | 非必须（CPU 推理） | — |

---

## 2. 安装步骤

### 2.1 获取源码

```powershell
# 进入项目目录（解压 源代码.zip 后）
cd topics\topic02_web_waf
```

### 2.2 创建虚拟环境（推荐）

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1        # Windows PowerShell
# source .venv/bin/activate          # Linux/macOS
```

### 2.3 安装依赖

```powershell
pip install -r requirements.txt
```

主要依赖：`flask`、`scikit-learn`、`xgboost`、`transformers`、`torch`（TinyBERT 语义轨）。

### 2.4 设置 Python 路径

```powershell
# Windows PowerShell
$env:PYTHONPATH="src"

# Linux/macOS
export PYTHONPATH=src
```

## 4. 启动 Web 服务（可运行程序）

### 4.1 方式一：统一 CLI（推荐）

```powershell
python scripts/iga_system.py serve
```

### 4.2 方式二：直接启动

```powershell
python run.py
```

### 4.3 访问地址

| 页面 | URL | 说明 |
|------|-----|------|
| 导航首页 | http://127.0.0.1:5000/ | 六页功能入口 |
| 实时监控 | http://127.0.0.1:5000/static/p1_monitor.html | 告警流 |
| 检测详情 | http://127.0.0.1:5000/static/p2_detail.html | 高亮定位 |
| 综合大屏 | http://127.0.0.1:5000/static/dashboard.html | ECharts 可视化 |

> 首次启动加载 TinyBERT 约需 30~60 秒，之后单条检测 P50 ≈ 2.9 ms。

### 4.4 修改监听地址（VPS 演示）

编辑 `configs/default.yaml`：

```yaml
server:
  host: "0.0.0.0"    # 允许外网访问
  port: 5000
```

## 5. API 调用示例

### 5.1 健康检查

```powershell
curl http://127.0.0.1:5000/api/health
```

### 5.2 单条恶意检测

```powershell
curl -X POST http://127.0.0.1:5000/api/detect `
  -H "Content-Type: application/json" `
  -d "{\"method\":\"GET\",\"url\":\"http://test/login?id=1+union+select+1,2--\"}"
```

返回关键字段：

```json
{
  "detection": {
    "is_malicious": true,
    "label": "SQLi",
    "confidence": 0.87,
    "latency_ms": 3.2
  },
  "explanation": {
    "highlight_html": "...<mark>union select</mark>...",
    "natural_language": "检测到 SQL 注入特征..."
  }
}
```

### 5.3 混淆变种生成

```powershell
curl -X POST http://127.0.0.1:5000/api/obfuscate `
  -H "Content-Type: application/json" `
  -d "{\"payload\":\"1' OR 1=1--\",\"attack_type\":\"SQLi\",\"count\":3}"
```

### 5.4 命令行单条检测

```powershell
python scripts/detect.py --url "http://x/test?p=1%2527%20union%20select" --json
```

## 6. 完整训练与评测流程（可选）

若需从零重建模型：

```powershell
# 1. 重建数据集（需 CSIC 原始数据）
python scripts/iga_system.py dataset

# 2. 训练 RF + TinyBERT
python scripts/iga_system.py train --epochs 5

# 3. 构建持续学习缓存
python scripts/iga_system.py build-cache --per-class 30
python scripts/iga_system.py expand-cache --max-rows 500

# 4. 全量诚实评测
python scripts/iga_system.py evaluate
# 结果写入 results/v2_exp1_overall.json

# 5. 实验套件 E2~E8
python scripts/iga_system.py experiments --experiments all
```

一键流水线：

```powershell
python scripts/iga_system.py pipeline
```

## 9. 常见问题

| 问题 | 解决方案 |
|------|----------|
| `ModuleNotFoundError: iga_guard` | 设置 `PYTHONPATH=src` |
| 首次请求很慢 | 等待 TinyBERT 加载完成 |
| `torch` 安装失败 | 使用 CPU 版：`pip install torch --index-url https://download.pytorch.org/whl/cpu` |
| 端口 5000 被占用 | 修改 `configs/default.yaml` 中 `server.port` |
| 评测指标与报告不一致 | 确认使用 `results/v2_exp1_overall.json` 最新文件 |
