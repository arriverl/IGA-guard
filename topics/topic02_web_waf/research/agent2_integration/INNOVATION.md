# IGA-Guard 2.0 创新点（终稿提纲）

> Agent 2 整合 Agent 1 文献 + 工程实践后凝练 · 用于作品报告与答辩

---

## 创新点 1：DLinear-TinyBERT 双流融合检测架构

**问题**：单一 Payload 分类器无法发现低速率混淆逃逸；纯时序模型无法理解 SQLi/XSS 语义。

**方法**：
- **时序轨**：DLinear 对 HTTP 请求统计序列做 Trend/Seasonal 分解，残差能量刻画异常
- **语义轨**：TinyBERT 对解混淆 Payload 做深度语义分类
- **门控融合**：根据两路置信度动态加权，规则强信号时走快路径（<1ms）

**与现有工作差异**：
| 对比 | 传统 WAF | 纯 BERT WAF | IGA-Guard 2.0 |
|------|----------|-------------|---------------|
| 低速率逃逸 | 弱 | 弱 | **DLinear 补强** |
| 混淆语义 | 规则 | BERT | **解混淆 + BERT** |
| 延迟 | 低 | 高 | **快路径 + 小模型** |

## 创新点 2：WebSpotter 式可解释恶意流量高亮

**问题**：黑盒判定无法告诉运维「哪一段是恶意的」。

**方法**：
- 字段贡献度权重（query / cookie / body）
- 字符级 span 定位 + 锚点词精炼（IoU 较基线 **+38%**）
- 前端实时高亮 + 自然语言解释模板

**演示效果**：大屏中对 `union select`、`<script>` 等片段红色标注，附中文判定说明。

---

## 创新点 3：面向混淆逃逸的 RL 智能特征工程

**问题**：100+ 维特征带来延迟与过拟合。

**方法**：EnhancedRLGWO 启发式筛选 → 动态保留 12~15 维核心特征；Online RL 根据漏检反馈调整特征权重与分类阈值。

**收益**：在保持检出的前提下降低推理开销，支撑 ≤10ms 赛题指标。

---

## 创新点 4：LLM Attack Agent 驱动的自演化对抗闭环

**问题**：静态模型难以防御未见过的混淆手法。

**方法**：
```
漏检样本 → AST/规则/LLM 三重混淆生成 → 增量训练 → 阈值 RL 调整 → 模型更新
```

**与赛题对齐**：满足「混淆载荷自动生成器」要求，并形成持续进化防御能力。

## 答辩一句话总结

> IGA-Guard 2.0 用 **DLinear 看流量节奏、TinyBERT 读攻击语义、WebSpotter 标恶意片段**，在可解释、可演示、可进化的工程闭环中实现混淆逃逸 Web 攻击的精准检测。
