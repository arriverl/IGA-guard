# 题目 3：基于 Transformer 架构的异常行为智能检测

## 题目背景

传统行为检测误报高、适应性差。Transformer 适合对长序列行为建模，捕捉复杂依赖与微妙偏差。

## 任务要求

### 1. 异常行为检测

- 输入：一个月主机行为日志（进程、网络、文件、注册表等）
- 输出：异常告警

### 2. 可解释性

每个告警需包含：

- 自然语言解释或关键特征标注
- 触发异常的行为片段
- 原始日志证据

## 挑战方向（二选一）

| 方向 | 目标 |
|------|------|
| 高精度 | 误报率 ≤20% 下最大化检出覆盖率 |
| 轻量化 | 检出率 ≥80%，无 GPU 可运行，资源占用最小 |

## 技术方案建议

### 数据预处理

```
原始日志 → 按主机/用户/session 分组
        → 事件序列化 (event_type, object, timestamp_delta)
        → 固定长度窗口 / 变长 padding
        → Token embedding
```

### 模型选型

| 方案 | 说明 |
|------|------|
| 轻量 Encoder | 2~4 层 Transformer，适合轻量化方向 |
| 预训练 + 微调 | BERT/RoBERTa 编码事件序列 |
| Anomaly Transformer | 注意力差异作为异常分数 |

### 可解释性

- 注意力权重 Top-K 事件 → 模板生成自然语言
- SHAP / Integrated Gradients 补充

## 及格线

- 标注异常检出率 ≥70%
- 误报率 ≤20%
- 全量日志稳定分析完成

## 评价标准

| 维度 | 权重 |
|------|------|
| 检测有效性 | 50% |
| 模型应用有效性 | 20% |
| 性能指标 | 15% |
| 可解释性与创新性 | 15% |
