# 题目 1：面向对话场景的大模型输入/输出违规内容过滤系统

## 题目背景

大语言模型在聊天、问答、客服等场景广泛使用，用户输入或模型输出可能出现低俗、暴力、广告、谣言等违规内容。工业界入门级方案以关键词匹配、语义规则、轻量文本分类为主。

## 功能要求

### 1. 违规词库与规则

- 四类：色情、暴力、广告、敏感话术
- 支持手动添加、删除、导入

### 2. 双层过滤

| 层级 | 方法 |
|------|------|
| 第一层 | 关键词 + 正则，快速拦截 |
| 第二层 | 开源预训练文本分类模型，语义二次判断 |

### 3. 分级处理

| 级别 | 处理 |
|------|------|
| 高违规 | 直接拦截，不转发大模型 |
| 低风险疑似 | 脱敏替换后送入大模型 |
| 正常 | 正常请求并返回 |

### 4. 输出二次校验

对大模型回答再次检测，违规则屏蔽并替换为标准合规话术。

### 5. 辅助功能

- 请求日志：检测结果、处理方式、时间
- 统计：每日违规数、类型占比

## 技术方案建议

```
用户输入 → [L1 规则] → 高违规? → 拦截
                ↓ 否
         [L2 分类模型] → 疑似? → 脱敏 → LLM API
                ↓ 否
              LLM API → [输出校验] → 返回/替换
```

### 推荐技术栈

- **后端**：FastAPI / Flask
- **LLM**：Qwen-Tiny、ChatGLM 精简版 API（或 Ollama 本地）
- **分类模型**：`hfl/chinese-roberta-wwm-ext` + 微调，或 `text2vec` + 轻量分类头
- **词库**：JSON/YAML + 管理 API
