## 2. 分题深度分析

### 题目 1：大模型违规内容过滤

**优势**

- 功能模块清晰：词库 → 双层检测 → 分级处理 → 输出校验 → 日志统计
- 入门友好，适合 4 人分工明确
- 演示效果好：对话界面即时可见拦截与脱敏

**风险**

- 误判率 ≤5% 需要精细调参
- 依赖开源 LLM API 稳定性

**适合队伍**：编程基础扎实、希望做完整 Web 产品

---

### 题目 2：Web 攻击载荷混淆逃逸检测

**优势**

- 与 ai-pentest-challenge 高度相关
- 技术深度足，实验充分

**风险**

- ≤10 ms 延迟约束苛刻
- 模块多、工程量大

**适合队伍**：有 Web 安全、CTF 经验

---

### 题目 3：Transformer 异常行为检测

**优势**

- 官方提供数据集
- Transformer + 可解释性是热点

**风险**

- 日志预处理复杂
- 训练周期长

**适合队伍**：有 GPU、熟悉序列建模

---

### 题目 4：大模型越狱自动化测试

**优势**

- 与 LLM 安全研究方向一致
- 报告可视化效果好

**风险**

- 成功率波动大
- API 成本
- 答辩需强调防御研究

**适合队伍**：熟悉 Prompt 工程

---

### 题目 5：DNS 隐蔽隧道检测（推荐）

**优势**

- 指标最清晰：85%/10%/5s/2工具
- 传统 ML 即可达标
- 合规风险低
- 可复用 cybersecurity_ml 流水线

**风险**

- CDN 场景易误报
- 需自建隧道流量

**适合队伍**：偏好流量分析 + 机器学习

## 3. 选题决策树

```
是否有稳定 GPU 且熟悉 Transformer？
├─ 是 → 考虑题目 3
└─ 否 → 是否擅长 Web 安全？
    ├─ 是 → 考虑题目 2
    └─ 否 → 是否做 LLM 相关？
        ├─ 是 → 题目 1 或 4
        └─ 否 → 推荐题目 5
```

**备选**：题目 2（Web 强）或题目 4（LLM 安全积累）
